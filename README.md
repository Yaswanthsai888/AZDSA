# AZDSA — EPI Python Solutions

This repository is a study and solution collection for the book "Elements of Programming Interviews in Python" (EPI Python).

## Structure

- `solutions/` - Python solution files organized by topic or chapter.
- `notes/` - study notes, patterns, algorithm ideas, and book summaries.
- `daily/` - daily progress notes, learning log, and short journal entries.
- `tests/` - test cases and validation scripts for solution files.

## Recommended workflow

1. Create a new file under `solutions/` for each problem or chapter.
2. Add comments and explanation in the same file or in `notes/`.
3. Use `daily/` for short progress notes and goals each day.
4. Commit frequently with clear messages, e.g. `Add binary tree traversal solution`.
5. Push to GitHub daily so your progress is visible and backed up.

## How to organize solutions

Use descriptive file names and folders, for example:

- `solutions/arrays/` for array-based problems
- `solutions/linked_lists/` for linked list problems
- `solutions/trees/` for binary tree problems
- `solutions/graphs/` for graph problems
- `solutions/dynamic_programming/`

## GitHub notes

- Keep the repository clean: code under `solutions/`, notes under `notes/`, and daily logs under `daily/`.
- Use issues or GitHub Projects if you want to track chapters and tasks.
- Add a short note in `daily/` whenever you solve a new problem or learn a new pattern.
